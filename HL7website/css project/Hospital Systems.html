Hospital Information System

HIS

Hospital information systems provide a common source of information about a patient's health history, and doctors schedule timing.
The system has to keep data in a secure place and controls who can reach the data in certain circumstances.

Clinical Support Systems

CIS

Clinical Information System

A clinical information system (CIS) is an information system designed specifically for use in the critical care environment, such as in an Intensive Care Unit (ICU). It can network with the many computer systems in a modern hospital, such as pathology and radiology.

LIMS

Laboratory Information Management System

A laboratory information system (LIS) is a software system that records, manages, and stores data for clinical laboratories. An LIS has traditionally been most adept at sending laboratory test orders to lab instruments, tracking those orders, and then recording the results, typically to a searchable database.

RIS

Radiology Information System

A radiology information system (RIS) is a networked software system for managing medical imagery and associated data. A RIS is especially useful for tracking radiology imaging orders and billing information, and is often used in conjunction with PACS and VNAs to manage image archives, record-keeping and billing.

Administrative Systems

FIS

Financial Information System

A financial information system is an organized approach to collecting and interpreting information, which is usually computerized. A well-run financial information system is essential to any business, since managers need the resulting information to make decisions about how to run the organization.

HRIS

Human Resources Information System

A human resources information system (HRIS) is a software solution that maintains, manages, and processes detailed employee and patient information and human resources-related policies and procedures.

MMIS

Materials Management Information System

Materials management information system (MMIS) is a software suite packaged as an integrated offering to meet materials management, human-resources and back-office needs. At a minimum, MMISs should be designed to interface readily with other mission-critical information systems in the enterprise.
